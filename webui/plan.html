Web UI - plan

Course building planning

Week by week:

  1. Feb 24~28: Gone researching for a usable training platform.

    We need a platform in order to

    • normalize training format (introducing the less possible restrictions)
    • keep track of student's actions (what parts of the course they used, when, for how long),
    • to keep track of grading actions like quizzes and exams,
    • to be able to automate exams and immediately return the result to the student,
    • to control course dependencies, ...

      Achievements:

      Moodle is succesfully installed and running, and we have a step-by-step guide to repeat the installation. See the notes.

      Also, Moodle was succesfully discarded. It suffers from two main issues:

      • It's outdated. It's request-response cycles with whole pege refresh are so 1999, and
      • The UI is terrible. Having globers create and take courses on suan a UI means assuming a recurrent extra-cost, as compared to doing the same on a usable UI that I haven't identified yet.
  2. Mar 5~7 (3 days): Assess the available course material in the Web UI track by
    • taking all exams
    • accessing and partially reading the current course material
    • taking notes

    The purpose is to rescue the valuable parts of what we already have (maybe fixing a few issues)

  3. Mar 10~14: Create the scheleton of a renewed Web UI track:
    • Define explicitly the purpose
    • Define explicitly the intended target
    • Enumerate the syllabus items
    • Split in weekly minicerts
    • With assesment capabilities
    • Automating quizzes
    • Filter out the absolute beginners by redirecting them to a basic course elsewhere, request a statement of acomplishment to the student if available (outsourcing!)
