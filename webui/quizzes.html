Quizz types

In order to better assess the glober's knowledge we need good quality exams, made up of a variety of formats like the ones enumeated below.

  1. One answer out of several (radios)

    Which of the following is true?

  2. Zero or more true statements, out of set (checkboxes)
  3. Match enumerated answers with their corresponding questions with DnD or selecting the question and then clicking the answer.
  4. Enter words or phrases, taken from a list, no repetition. Like: "Name the Scandinavian countries". Compare plain or using regexps.
  5. Enter a (rather) exact text (possibly more then one), with some elasticity
    1. with or without quotes
    2. normalize blank space
    3. case-insensitive (optional)
    4. strip accents
    5. match a regular expression

    This can be achieved by massaging the text and/or providing a set of valid answers.

    It should be possible to set a "never mind" answer for a particular question that accepts any answer.

    Also, consider symbolic answers, like values from system variables or collected from previous quizzes.

    Allow for checking if the values are numeric and doing some calculations on then in order to check the correctness.

  6. Allow entering some JavaScript or Java code and to run it for example in Node.js, somehow sandboxed, and use the output as the result to be checked, maybe the console log, or the content of a specific variable.

 

The exams system

There is a questions repository. Each question has its own answer(s). The questions are tagged with subjects.

An exam is made up of one or more question sets, each question set is made up of questions taken from a subject. As a question can belong to more than one subject it may appear twice in the same exam.

From each subject, an exam includes a fixed number of questions or points (its size). Some of the questions are fixed and some randomly chosen to complete the exam size.

The questions are numbered, the question number is its key in the database and its id in the exam page (prefixed with a "q").

The answers of a question are stored with the question, numbered from 1 on, and an answer's number never changes. It's possible for the sequence to have holes, no problem.

In the page que answers have an id made up of the question id concatenated with a hyphen "-" and the answer number, no left zeros. For example answer 7 of question 1234 has id="q1234-7". The name="" attribute is equal to the id when possible. The value="" of the answers is equal to the answer number.
