Esto es un análisis genérico sobre la capacitación para los globers del bench, y sugerencias a implementar para obtener mejores resultados con gasto mínimo.

Analizado de nuevo, el sistema de exámenes ahora me parece más adecuado para Career que para Staffing, por que el objetivo de Staffing es rotar la gente rápidamente, mientras que Career maneja educación a largo plazo, como por ejemplo un tester que quiere ser developer.

Sin embargo algunas de las ideas pueden ser aplicables.

 

Visión

Handbook

Cuando un glober entra al bench, a primera hora del primer día recibe un email con un link a un documento "Bench Handbook" con instrucciones detalladas, que está obligado a leer y conocer.

El Handbook le comunica sus obligaciones y le informa sobre la visibilidad de la actividad que desarrolle mientras no está asignado.

En el caso normal el Handbook lo instruye para que comience a rendir exámenes.

Staffing mantiene el Handbook actualizado. Contiene links a los recursos que el glober necesita, para que pierda la menor cantidad de tiempo inicial. Puede ser gente que viene de trabajar intensamente en la terminación exitosa de un proyecto, y no queremos que pierda la velocidad que trae.

Exámenes

Instruído por el Handbook el glober entra al Sistema de Exámenes. El objetivo de este sistema es registrar el conocimiento de los globers desasignados para poder venderlo.

Los exámenes son independientes de los cursos: no nos importa cómo adquieren el conocimiento, nos importa si lo tienen o no.

Los exámenes están organizados por temas: Web UI, Web Backend, Testing, Java programming, whatever.

Cada tema es un arbol con branches y levels. Por ejemplo Web Backend tiene branches Java, .NET, Node.js.

El sistema registra el avance del glober; un glober que vuelve al bench al final de un proyecto vé el mismo estado de la última vez que estuvo en el sistema.

Los globers pueden seguir aprobando exámenes aunque no estén en el bench.

La construcción de un set de exámenes para verificar la cantidad y variedad de conocimientos de la actualidad es un trabajo enorme, que no tiene fin por la volatilidad de las tecnologías.

Cursos

Time tracker

Los globers del bench deben completar el Time Tracker.

Esto es para que perciban que ese tiempo le importa a Globant.

En "notes" deben anotar el curso que están haciendo y el avance, por ejemplo "Selenium intro - day 4".

Al comienzo de cada capacitación reciben el texto que deben cargar cada día.

Training & Courses

La capacitación está orientada a gente autodidacta y con ganas de aprender o profundizar sus conocimientos sobre Web UI. Esta iniciativa busca generar conocimiento sobre Web UI dentro de Globant y no ser una certificación de conocimiento.

Estas capacitaciones son actividades de Globant para que los Globers podamos progresar en conocimientos técnicos y, a la vez, para mejorar el valor de la oferta de Globant a nuestros clientes.

Formato

Son módulos chicos, relacionados entre sí por un arbol de dependencias.

Algunos módulos son específicos de un track en particular, mientras que otros son genéricos y se conectan donde son necesarios. Por ejemplo, para hacer un módulo de JavaScript es necesario tener una base de programación independiente del lenguaje.

The students must pass all dependencies tests. They have the option of taking or skipping the course but approving the test is a must. This way we can be sure about the general knowledge level, independently of the student's self-esteem.

If a student opts for skipping the course, goes straight to the test and doesn't pass then he must take the course, re-taking the test is not allowed.
